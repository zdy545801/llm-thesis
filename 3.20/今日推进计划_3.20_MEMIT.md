# 今日推进计划（3.20）- MEMIT

## 1. 今日目标
在 Colab 完成 MEMIT 连续编辑主线首跑，并产出可用于论文对照的结果文件：
- `results/MEMIT/MEMIT_stress_10rounds_YYYYMMDD/round_*.json`
- `results/MEMIT/MEMIT_stress_10rounds_YYYYMMDD/drift_metrics.json`
- `results/MEMIT/MEMIT_stress_10rounds_YYYYMMDD/drift_curve.png`

## 2. 最短执行顺序（Colab）
1. 打开 `COLAB_FT_R1.ipynb`（或 `COLAB_FT_R2.ipynb`）。
2. 依次运行环境初始化与依赖安装单元。
3. 在“写 hparams”单元把算法改为 `MEMIT`，文件名改为 `MEMIT_gpt2xl.yaml`。
4. 在“运行连续编辑”单元将 `--hparams` 指向 `MEMIT_gpt2xl.yaml`，其余参数先保持：`rounds=10, step=6`。
5. 运行“检查轮次结果”单元，确认 `round_0 ~ round_10` 都存在。
6. 运行“计算漂移指标”和“画图”单元。
7. 打包下载 `rounds + drift_metrics.json + drift_curve.png`。
8. 本地归档到 `results/MEMIT/MEMIT_stress_10rounds_YYYYMMDD/`。

## 3. 建议参数（首跑）
- 模型：`gpt2-xl`
- 编辑集：`data/edits_bias_stress_60.json`
- 评测集：`data/crows_pairs_sample.jsonl` + `data/bbq_sample.jsonl`
- 轮次：`rounds=10`
- 每轮增量：`step=6`
- 连续编辑：`sequential_edit=True`

## 4. 结果检查标准
- `round_*.json` 按轮次连续存在（无缺号）。
- `drift_metrics.json` 中 `rounds` 列表长度与轮次数一致。
- `drift_curve.png` 可见风险曲线与 FDR 曲线。

## 5. 常见阻塞与处理
1. 显存不足（OOM）：先降为 `rounds=6, step=4` 验证可跑，再恢复正式设置。
2. 运行过慢：MEMIT 首轮可能会计算统计量，属于正常现象。
3. Colab 会话冲突：先在 `Manage sessions` 里关闭旧会话再开新会话。
